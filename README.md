# Тестовое задание frontend developer

## Описание

Необходимо используя Github GraphQL API (https://docs.github.com/ru/graphql) создать frontend приложение, которое состоит из двух страниц.

## Страницы приложения

### 1. Главная страница

#### Ключевые элементы:

- Поле для поиска репозиториев
- Список репозиториев
- Cursor-based paginator

#### Функциональность:

- При вводе текста происходит поиск по названию среди всех репозиториев Github
- При пустом поле показывается список репозиториев текущего пользователя
- Поиск осуществляется на стороне API
- При клике на название репозитория происходит переход на карточку репозитория

#### Структура элемента списка:

`[Название репозитория] - [кол-во звёзд] - [дата последнего коммита] - [ссылка на Github]`

#### Пагинация:

- Курсорная пагинация, серверная.
- С кнопками "Next" и "Previous", которые позволяют перемещаться по страницам
- Нужно сделать так, чтобы при перезагрузке страницы состояние фильтров (поиск и страница) сохранялось и использовалось для первоначального запроса

### 2. Карточка репозитория

#### Структура карточки:

- Название репозитория, количество звёзд, дата последнего коммита
- Фото владельца репозитория (если есть)
- Nickname владельца с ссылкой на профиль
- Список используемых языков
- Краткое описание репозитория

## Технические требования

### Обязательные технологии:

- Vite
- React Router
- TypeScript
- React
- GraphQL
- FSD (Feature Sliced Design) архитектура

### Стейт-менеджер:

- Zustand
- Данные должны храниться в стейт-менеджере, но только если это действительно необходимо

### Ограничения:

- Нельзя использовать готовые UI библиотеки
- Вся верстка должна быть написана самостоятельно

## Результат

- Ссылка на личный репозиторий
- Желательно: ссылка на работающее приложение

## Примечание

Будет большим плюсом, если настроите типизацию в проекте так, чтобы она была максимально удобной для работы с GraphQL. Например, используя `graphql-codegen`

Дизайн может быть минималистичным, но функциональным. Главное - удобство использования в рамках описанных сценариев.
